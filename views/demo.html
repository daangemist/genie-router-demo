genie-router

A generic platform that will route commands and conversations from voice or text-based clients to 3rd party backends. This allows you to interact to different services ('brains') via your own defined set of input channels.

View on Github | Follow on Twitter | Show introduction

Hi, this demo will show you some of the features of genie-router. It uses the HTTP API plugin to send your input to the router. Additional channels are set up that will interact with the same setup, in Telegram and Facebook Messenger

Besides clients, genie-router also has the concepts of Brains. These are plugins for services than process input and generate an output to display. In this demo two brains are integrated: rivescript and dialogflow. Rivescript is a scripting engine, the demo uses an ELIZA implementation by the author of rivescript. The dialogflow agent is trained to answer some generic questions about genie-router.

How is the distinction made between the two brains? The mentions plugin allows you to invoke different brains by mentioning their plugin name, or a configured alias. For this demo the dialogflow agent is dubbed genie. The rivescript plugin is configured as the default brain, if no brain is mentioned in the input.
